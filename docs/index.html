HISTORIA



Rasmus Lerdorf

El desarrollo de PHP comenzó en 1994, cuando Rasmus Lerdorf escribió varios ficheros binarios Common Gateway Interface (CGI) en el lenguaje C, los cuales fueron concebidos para gestionar su página web personal. Extendió sus funcionalidades hasta el tratamiento de formularios web y la comunicación con una base de datos, y llamó a ésta implementación “Personal Home Page Tools” o “PHP-Tools”.


El 8 de Junio de 1995 se lanzó oficialmente la primera versión de PHP, conocida como la versión 1.0. Esta versión, aunque aún no era un lenguaje de programación totalmente independiente, ya contenía varias características que vemos en PHP hoy en día: variables tipo Perl (de ámbito dinámico y estático), tratamiento de formularios web e incrustación del código en un documento HTML.


Zeev Suraski
Andi Gutmans

El proyecto, aunque empezó desarrollándolo Lerdorf a título personal, fué revisado en 1997 por Andi Gutmans y Zeev Suraski, quienes cambiaron el analizador sintáctico (parser) y terminaron por definir PHP como un lenguaje de programación independiente, lanzando la versión 3.0 . A partir de entonces, las siglas PHP pasaron a significar “Preprocesador de hipertexto”.


El proyecto crecía en popularidad con el paso del tiempo. En 1998 Gutmans y Suraski cambiaron el núcleo de PHP y crearon el Zend Engine (Motor Zend). Éste motor mejoraba la ejecución de aplicaciones complejas y aportaba modularidad al código fuente. En 2004, se añadieron funcionalidades del paradigma orientado a objetos, como un modelo de datos orientado a objetos.


Las versiones se fueron sucediendo, convirtiendo PHP en una tecnología cada vez más robusta y potente. Actualmente, la versión más reciente de PHP es la 7.2 . Sus características se verán en profundidad en el apartado “Características de PHP”.



Versión Año Descripción
1.0 1995 Se producen varios ficheros CGI para la gestión de un formulario y una base de datos de una página web. Es la primera vez que se utiliza el nombre PHP.
2.0 1997 A partir de ésta versión, podemos considerar por primera vez PHP como un lenguaje autónomo.
3.0 1998 El desarrollo del proyecto crece en desarrolladores. Reescribieron el parser o analizador sintáctico. Se considera el nacimiento del PHP que conocemos hoy en día.
4.0 2000 Cambiaron el núcleo, produciendo el Motor Zend.
5.0 2004 Se desarrolló el Motor Zend 2. Se añadieron características de la orientación a objetos.
6.0 --- No terminó de desarrollarse, si no que las mejoras planteadas se incluyeron en las versiones 5.3 y 5.4. Se pretendió integrar algunas características Unicode al lenguaje.
7.0 2015 Se produjo el Motor Zend 3, aumentando la eficiencia y la velocidad de ejecución de los procesos.


CARACTERÍSTICAS



PHP es un lenguaje de script del lado del servidor, usado para desarrollar páginas web estáticas, dinámicas o aplicaciones web. Los scripts en PHP sólo se pueden interpretar en un servidor que tenga PHP. Las máquinas cliente sólo pueden acceder al fichero PHP a través de un navegador. Un fichero en PHP contiene etiquetas PHP y termina con la extensión “.php”.


¿Qué es un lenguaje de script?


Un script es un conjunto de instrucciones que son interpretadas en tiempo de ejecución. Un lenguaje de script es un lenguaje que interpreta scripts en tiempo de ejecución. Los scripts suelen estar embebidos en otros entornos de software. El objetivo de los scripts suele ser mejorar el rendimiento o realizar tareas rutinarias para otra aplicación.


Comparación entre un lenguaje de programación y un lenguaje de script


Lenguaje de programación Lenguaje de script
Tiene todas las características necesarias para el desarrollo integral de aplicaciones Usado principalmente para procesos rutinarios
El código debe ser compilado antes de ser ejecutado El código suele ser ejecutado sin haber sido compilado (intérprete)
No necesita ser embebido en otros lenguajes Tiende a ser embebido dentro de otras tecnologías de desarrollo

PHP es un script del lado del servidor, lo que quiere decir que se interpreta en el mismo, al contrario que JavaScript, que es un script interpretado por el navegador web en el lado del cliente. Ambos lenguajes pueden ser incrustados en un documento HTML.


PARADIGMAS DE PHP


Imperativo Procedural Funcional OOP Reflexivo

IMPERATIVO



Un lenguaje de programación imperativo es aquel que utiliza sentencias para cambiar el estado del programa. De una manera muy parecida al modo imperativo en los lenguajes naturales, un programa imperativo se compone de comandos u órdenes destinadas a la computadora. El término se utiliza para diferenciarlo de los lenguajes declarativos, los cuales se centran más en el resultado esperado que en los procesos concretos para obtener dicho resultado.


PROCEDURAL



Un lenguaje procedural es un tipo de lenguaje imperativo, donde la ejecución del programa está basada en procesos. Un proceso no es más que una secuencia ordenada de instrucciones que se ejecutan de una manera predefinida. Cuando el proceso se termina de ejecutar, el programa retorna a la línea anterior a la llamada al proceso (la dirección de retorno). Los procesos se pueden invocar desde cualquier punto del programa, e incluso se pueden llamar unos a otros.


FUNCIONAL



El paradigma funcional de la programación se basa en el uso de funciones matemáticas, en lugar de imperativas. Éstas, por su naturaleza matemática, evitan el cambio de estado o la mutación de datos. De ésta manera, en el paradigma funcional, una función a la que se le pasan los mismos argumentos siempre tendrá el mismo comportamiento, a diferencia de una función imperativa, la cual tiene efectos secundarios en otros cálculos y puede comportarse de manera distinta en un momento u otro de la ejecución.


La programación funcional es un tipo de programación declarativa


ORIENTADO A OBJETOS



La orientación a objetos consiste en la encapsulación de la información en objetos, una unidad compleja de información. Cada objeto contiene datos en forma de atributos y código ejecutable en forma de métodos o funciones. Los objetos trabajan enviándose mensajes unos a otros. De ésta manera, la información está mejor organizada y los programas son más fáciles de entender y manipular.


REFLEXIVO



En informática, la reflexión es la capacidad de un programa de analizarse a sí mismo y modificar su propia estructura o comportamiento (concretamente los valores, metadatos, propiedades y funciones) en tiempo de ejecución. Este concepto sirve para crear código muy potente, y se considera una práctica de la metaprogramación.



FRAMEWORKS



Laravel Symphony Zend Framework Phalcon FuelPHP

LARAVEL


Laravel es un framework de PHP relativamente nuevo, pero indiscutiblemente el más popular hoy en día. Es gratuito y de código abierto, y se utiliza en el desarrollo web en PHP siguiendo el patrón de arquitectura MVC (modelo-vista-controlador). Aprender Laravel es fácil pues está totalmente documentado, y además su página web oficial ofrece muchos videotutoriales. Algunas de sus características son: permite desarrollar aplicaciones más fácil y rápido, un sistema de archivos con un gestor de dependencias dedicado y utilidades que ayudan en el despliegue y el mantenimiento.


Incluye Symphony como dependencia propia.


SYMPHONY


Symphony es tanto un framework para el desarrollo web en PHP como un conjunto de componentes reusables de PHP. Sus librerías permiten una fácil integración con otras aplicaciones y con frameworks de front-end como AngularJS. Cuando se trata de desarrollar proyectos de gran tamaño, Symphony puede ser de gran utilidad. Además, está respaldado por una amplia comunidad de desarrolladores, lo cual garantiza su estabilidad y robustez.


Symphony fué pionero en desarrollar una barra de herramientas de desarrollador que permite detectar y corregir errores en fases tempranas del desarrollo. Ésta barra es de uso generalizado en otros frameworks de hoy en día.


ZEND FRAMEWORK


Zend es un framework muy estable y robusto con un amplio rango de opciones de configuración. Reúne un grupo de funcionalidades como un editor intuitivo de arrastrar y soltar con soporte para front-end, herramientas de programación criptográfica, un instalador de bases de datos y herramientas para debuguear instantáneamente vía online. Tiene colaboraciones con IBM, Microsoft, Google y Adobe.


No es recomendado para proyectos pequeños, pues es difícil de aprender.


PHALCON


Phalcon está escrito en C y C++, lo que lo convierte en el framework PHP más rápido de todos, pues está optimizado a muy bajo nivel. De ésta manera, el consumo de recursos por parte de Phalcon es mínimo. Algunas de sus funcionalidades son un auto-loader, un gestor de recursos, características de seguridad, portabilidad, y una documentación muy amplia. A diferencia de Laravel, Phalcon no es de código abierto.


FUELPHP


FuelPHP es un framework PHP modular, flexible y full-stack que no sólo soporta el MVC (modelo-vista-controlador) si no también su vesión evolucionada, el HMVC, y ambos a nivel de arquitectura. FuelPHP añade además una clase opcional llamada “Presenter” que intermedia entre el controlador y la vista y cuya función es mantener la lógica necesaria para generar vistas.


El framework es capaz de gestionar la seguridad del proyecto gracias a un filtro de entrada y URIs y un codificador de salida. Además tiene su propio framework de autentificación.
